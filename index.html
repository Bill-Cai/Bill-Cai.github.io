DBSCAN - 原理、实现和并行化尝试

概念

DBSCAN（Density-Based Spatial Clustering of Applications with Noise），有噪声的基于密度聚类算法。

  • 将簇定义为具有足够高密度的区域；
  • 可以在有噪声的空间数据中发现任意形状的聚类。

关于DBSCAN的一些定义：

  • $E$邻域：对于给定对象，半径为Ε内的区域。
  • 核心对象：$E$邻域内样本点数大于等于$MinPts$的给定对象。
  • 直接密度可达：对于样本集合$D$，如果样本点$q$在$p$的$Ε$邻域内，并且$p$为核心对象，那么对象$q$从对象$p$直接密度可达。
  • 密度可达：对于样本集合$D$，给定一串样本点$p_1,p_2,…,p_n$，$p=p_1$，$q=p_n$，假如对象$p_i$从$p_{i-1}$直接密度可达，那么对象$q$从对象$p$密度可达。
  • 密度相连：存在样本集合$D$中的一点$o$，如果对象$o$到对象$p$和对象$q$都是密度可达的，那么$p$和$q$密度相联。

DBSCAN目的是找到密度相连对象的最大集合。

个人博客搭建 - 基于Hexo

搭建思路

  1. 安装Hexo
    1. 安装node.js环境（node，npm）
    2. 局部安装Hexo：npm install hexo
  2. 新建blog文件夹，并cd到blog文件夹下
  3. 通过npx hexo init初始化
  4. 本地启动hexo服务：npx hexo server或者简写为npx hexo s
  5. 访问默认的http://localhost:4000/网页查看

新建博客

  1. 创建md文件：npx hexo n "blog-file-name"
  2. 删除hexo生成的所有文档（其实就是清理public文件夹）：npx hexo clean
  3. 重新生成页面文档：npx hexo g
  4. 启动服务，访问博客网页

网页部署

  1. 注册GitHub账号，创建代码仓库，要求仓库名为<username>.github.io，因为GitHub Page解析的时候是找的这个名称

  2. 安装Hexo依赖：npm install --save hexo-deployer-git

  3. 修改配置文件_config.yml，在最下方的deploy部分添加git配置

  4. 部署（上传至GitHub）：npx hexo d
